# Wonder Trail Presentation Improvement Plan

## Current Feeling

The Alpha is playable, but the presentation still feels dry because:

- The forest is a CSS placeholder rather than illustrated world art.
- Search objects are emoji/CSS tokens, not cohesive game assets.
- Stage transitions are functional but not magical.
- Rewards are text-first and do not yet feel like discoveries.
- Xiaohang is visually rich, but the rest of the scene is much simpler.

## Goal

Make the Alpha feel more like:

> Xiaohang is taking the child into a living forest.

Not:

> A worksheet with a character beside it.

## Immediate Improvements

These can be done before final art:

1. Add a small story beat before each stage.
2. Make success feedback more visual, such as sparkle bursts or object pop animations.
3. Add a "forest book" reward screen instead of plain reward text.
4. Give the progress bar a more story-like form, such as stepping stones or forest path markers.
5. Add gentle background motion, such as drifting leaves or soft light movement.
6. Make the final treasure stage feel more special with a larger animation.

## Asset-Based Improvements

These need better visual assets:

1. Replace CSS forest with a portrait-friendly illustrated forest background.
2. Replace emoji objects with illustrated Search objects.
3. Add small foreground layers such as leaves, flowers, stones, and grass.
4. Create sticker art that matches the Search objects.
5. Create Xiaohang expression variants:
   - hello
   - curious
   - happy
   - thinking
   - celebrating

## Interaction Improvements

1. Correct tap:
   - object pops
   - sparkle appears
   - Xiaohang says a short positive line

2. Wrong tap:
   - small soft bounce
   - no red error
   - Xiaohang gently suggests trying again

3. Hint:
   - target area glows
   - Xiaohang leans in or points
   - voice gives one short clue

4. Reward:
   - sticker flies into forest book
   - stars count up softly
   - child sees what was added

## Sound Improvements

Keep free system voice for now, but improve:

- Shorter lines
- More playful wording
- Less command-like phrasing
- Clear pause between feedback and guidance

Future:

- Record fixed Xiaohang clips.
- Or use AI TTS for a consistent Xiaohang voice.

## Recommended Next Build

The next best improvement is:

> Add visual reward and success animations before adding more mechanics.

This will make the game feel more alive without expanding scope too much.
